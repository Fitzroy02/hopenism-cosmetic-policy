# MODERATION WORKFLOW

## Purpose

To ensure every product and seller is reviewed, escalated, and actioned in a consistent, traceable, and safety-first manner.

---

## I. Moderation Triggers

### Automated triggers

- Banned ingredient detected
- Restricted ingredient above threshold
- Medical/borderline claim detected
- Missing CPSR/PIF
- Incorrect or incomplete label
- High-risk category (peels, retinoids, devices)
- Seller Tier 1 attempting Tier 3 listing

### Human triggers

- Consumer complaint
- Adverse event report
- Suspicious seller behaviour
- Regulatory inquiry

---

## II. Moderation States

| State | Description |
|-------|-------------|
| **Pending Review** | Awaiting automated checks |
| **Flagged** | Automated checks found issues |
| **Under Human Review** | Moderator evaluating |
| **Approved** | Listing is safe and compliant |
| **Rejected** | Listing violates rules |
| **Suspended** | Temporarily removed |
| **Escalated** | Sent to Compliance Lead |

---

## III. Moderator Actions

- Approve
- Reject (with reason)
- Suspend
- Request documentation
- Request reformulation
- Escalate to Compliance Lead
- Notify Responsible Person
- Notify regulator (if required)

---

## IV. Audit Trail Requirements

Every moderation action must record:

- moderator ID
- timestamp
- action taken
- rationale
- snapshot of listing at time of action

---

## V. Escalation Path

1. Moderator
2. Compliance Lead
3. Governance Steward
4. External regulator (if safety risk is serious)
