# Seller Dashboard UX

This document defines concise, text-first UX requirements for the seller dashboard.

## Overview

- **Metrics:** active listings, pending reviews, rejected listings, compliance score.
- **Alerts:** action-required cards (missing CPSR, flagged claims, expiring documentation).

## Products

- **Table columns:** product name, status (Approved/Pending/Rejected/Suspended), risk level, last review date.
- **Actions:** edit, view compliance report, duplicate, withdraw.

## Compliance Centre

- **Status badges:** CPSR ✅/❌, PIF ✅/❌, Label ✅/❌, Claims ✅/⚠️.
- **Guidance links:** “Why was this rejected?” and “How to fix this”.

## Documentation Vault

- Upload CPSR, PIF, labels, ingredient lists.
- Maintain version history per document.

## Account & Qualification

- Display tier level (1/2/3).
- Show requirements to upgrade.
- Show verification status.
