# Onboarding

This guide helps new contributors work safely and consistently with repository policy.

## First Day Checklist

- Read the [Constitution](./constitution.md).
- Review [Governance](./governance.md) responsibilities.
- Study [Safeguards](./safeguards.md) checklist and red lines.
- Read [Culture](./culture.md) expectations.

## Contribution Flow

1. Choose a policy area and define the problem.
2. Draft changes in clear, testable language.
3. Include evidence and risk considerations.
4. Run through safeguards checklist.
5. Submit for governance review.

## Quality Standard

Every contribution should be:

- Understandable to non-specialists,
- Explicit about assumptions and limitations,
- Traceable to evidence or rationale,
- Versioned with meaningful changelog notes.
